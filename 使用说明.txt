═══════════════════════════════════════════════════════════════
  iMouse Pro 自动化工具 V2.0 - 使用说明
═══════════════════════════════════════════════════════════════

【前置要求】
─────────────────────────────────────────────────────────────
1. 必须先启动 iMouse 专业版内核（监听 localhost:9912）
2. 手机必须通过 USB 连接并在 iMouse 控制台显示"在线"
3. 首次运行：双击 install.bat 安装 Python 和依赖包
4. 日常启动：双击 start.bat
5. 工具启动后会自动连接内核并加载设备列表


═══════════════════════════════════════════════════════════════
  一、设备列表（左侧面板）
═══════════════════════════════════════════════════════════════

【刷新设备】
  重新从内核获取当前在线设备列表
  如果设备断开或新增，点此按钮同步

【全选 / 反选】
  勾选（[x]）或取消勾选设备
  勾选后，"发布到选中设备""选设备切号"会对勾选的设备生效

【设备列表列含义】
  V        — 勾选状态 [ ] 或 [x]
  自定义名  — 你在 iMouse 控制台设置的设备别名（如 01、02）
  设备名    — iMouse 内核给设备的编号
  型号      — iPhone 型号（如 iPhone 11）
  IP        — 设备 IP 地址


═══════════════════════════════════════════════════════════════
  二、一键发布（Tab 1：发布 TikTok 内容）
═══════════════════════════════════════════════════════════════

【基本流程】
  1. 选择内容类型：图片 / 视频
  2. 提供素材（两种方式选一）：
     方式 A - 单个素材文件：所有设备用同一个文件
     方式 B - 素材文件夹：按设备自定义名分子文件夹
  3. （可选）填写音乐 URL、标题、描述
  4. 点击"一键发布到所有在线设备"或"发布到选中设备"

【素材文件夹模式 - 重要】
  ⚠ 子文件夹名必须与设备的"自定义名"完全一致
  结构示例：
    D:\iMousePro\Shortcut\Media\
        ├── 01\                  ← 对应自定义名 "01" 的设备
        │   ├── video.mp4
        │   └── photo.jpg
        ├── 02\                  ← 对应自定义名 "02" 的设备
        │   └── video.mp4
        ├── 03\
        │   └── xxx.mp4
  工具会自动匹配每台设备对应的子文件夹

【发布参数说明】
  步骤间隔(秒)：3  — 每步之间的等待时间
  识别等待(秒)：15 — 图标识别最长等待时间
  设备间隔(秒)：2  — 串行模式下设备之间的间隔

【存草稿选项 - 新功能】
  ☑ 勾选"存草稿(不发布，保存为草稿)"后：
    - 完成标题和标签输入后，不点 Post
    - 改为点击 Drafts 按钮，将内容保存为草稿

【图片按索引选图 - 新功能】
  - 发布图片时不再自动选最后一张
  - 根据任务序号（1、2、3...）对应选择相册里的第 1、2、3 张图
  - 批量导入 Excel 时，第 N 行任务对应相册第 N 张图

【Excel 批量导入】
  点"导入 Excel 任务"后选择 .xlsx 文件
  Excel 格式（第一行表头）：
    devices | file | type | url | title | description | status | scheduled_time
  - devices: 设备自定义名，多个用逗号分隔（如 "01,02,03"）
  - file: 素材文件路径
  - type: picture 或 video
  - url: 音乐 URL（可选）
  - title: 标题
  - description: 描述标签
  可用 tiktok_data.xlsx 作为模板

【注意事项】
  ✓ 发布前请确保手机已安装 TikTok 并登录
  ✓ 发布过程依赖图标识别，横屏或异常界面会影响识别
  ✓ 如果发布卡住，点"停止发布"


═══════════════════════════════════════════════════════════════
  三、一键切号 / 选设备切号
═══════════════════════════════════════════════════════════════

【功能】
  对目标设备切换 TikTok 账号（当前账号 → 另一个账号）

【流程】
  HOME → 打开 TikTok → 个人页 → 三条杠菜单 →
  Settings and privacy → 滚动 → Switch account → 选另一个账号

【两个按钮区别】
  一键切号      — 所有在线设备都执行
  选设备切号    — 只对勾选的设备执行

【注意事项】
  ✓ 每台设备必须已登录至少 2 个账号（否则 Switch account 列表为空）
  ✓ 工具会自动识别活跃账号（带红勾✓）并点击另一个账号
  ✓ 切号会自动检测横屏并返回
  ✓ 多台设备并发执行


═══════════════════════════════════════════════════════════════
  四、一键养号 / 停止养号 - 新功能
═══════════════════════════════════════════════════════════════

【功能】
  模拟真实用户浏览 TikTok，滑视频 + 点赞，养号

【流程】
  Phase 1: 初始化（所有设备并发）
    - 按 Home 键回桌面
    - 打开 TikTok
    - 点击底部 Home 图标（For You 入口）
    - 向右滑到 For You 标签

  Phase 2: 无限批次循环
    - 每批次把所有在线设备随机分成 2-4 组
    - 组内设备并发，每台设备随机延迟 0-20 秒
    - 每台设备执行：
      1. 检测横屏（宽>高）→ 点(71,387)切换直播模式
      2. 从屏幕中心向上滑（切到下一个视频）
      3. 点赞（视频模式）或 双击屏幕中心（直播模式）
    - 批次之间间隔 10 秒

【控制】
  点"一键养号"开始 → 点"停止养号"结束
  养号期间可继续用工具其他功能，但请不要对同一设备同时执行其他操作

【注意事项】
  ✓ 会无限循环运行直到点"停止养号"
  ✓ 每批次会随机重新分组，模拟真实用户行为
  ✓ 每台设备有独立的直播模式状态跟踪
  ✓ 点赞按钮位置默认 (386, 469)，如果 TikTok 更新了界面可能失效


═══════════════════════════════════════════════════════════════
  五、上传（Tab 2：文件上传到手机）
═══════════════════════════════════════════════════════════════

【上传目标】
  ○ 上传到相册     — 图片和视频进相册 (Recents)
  ○ 上传到文件系统  — 任意文件到 iOS 文件 App（iOS 15+）

【上传方式 1：按自定义名分文件夹 - 推荐】
  ⚠ 与发布的文件夹模式相同，子文件夹名 = 设备自定义名
  每台设备自动上传对应子文件夹里的所有文件
  例：
    Media\01\ → 自定义名为"01"的设备
    Media\02\ → 自定义名为"02"的设备
    Media\03\ → 自定义名为"03"的设备
  注意：子文件夹名必须和"自定义名"列完全一致（大小写、空格都要一致）

【上传方式 2：统一文件上传】
  选中若干文件 → 上传到选中设备 或 一键上传到所有设备
  所有设备都上传同样的文件

【按钮说明】
  上传到选中设备    — 只上传到左侧勾选的设备
  一键上传到所有设备 — 上传到所有在线设备

【视频上传要求 - 重要】
  ✓ 支持格式：MP4、MOV、M4V（iOS 原生支持）
  ✗ 不支持：AVI、MKV、WMV、FLV（需先转成 MP4）
  ✓ 编码要求：H.264/H.265 + AAC
  ✓ 路径：Windows 本地绝对路径（如 D:\videos\test.mp4）
  ✓ 单文件上传超时：10 分钟
  ✓ 逐个文件上传（不批量），避免单个大视频拖垮整批

【相册/目标路径】
  留空 = 上传到"Recents"（相册默认位置）或文件系统根目录
  指定相册名 = 上传到该相册（如"我的素材"）
  文件系统路径需以 / 开头，如 /iMouse/videos/

【注意事项】
  ✓ 路径不能是网络路径或手机上的路径
  ✓ 文件名和路径可以包含中文
  ✓ 上传过程中不要断开 USB


═══════════════════════════════════════════════════════════════
  六、下载（Tab 3：从手机下载文件到 PC）
═══════════════════════════════════════════════════════════════

【下载来源】
  ○ 从相册下载     — 手机相册的照片和视频
  ○ 从文件系统下载  — 手机文件 App 的文件（iOS 15+）

【流程】
  1. 勾选一台设备（下载功能一次只对一台设备操作）
  2. 填相册名/路径（留空=默认相册/根目录），填数量
  3. 点"获取文件列表"
  4. 在列表中勾选要下载的文件（可 Ctrl+点多选）
  5. 点"下载选中文件"

【文件保存位置】
  相册下载：iMouse 内核目录\Shortcut\Media\
  文件下载：iMouse 内核目录\Shortcut\File\

【注意事项】
  ✓ 每次操作只能针对一台设备
  ✓ 下载后手机上的原文件不会删除
  ✓ 相册下载时，文件名=手机相册里的名字
  ✓ 下载完成提示后，去对应文件夹查看

【功能】

  2. 点"查看相册"或"查看文件系统"

═══════════════════════════════════════════════════════════════
  八、通用问题排查
═══════════════════════════════════════════════════════════════

【连接失败】
  - 确认 iMouse 专业版内核已启动（图标在任务栏）
  - 确认端口 9912 没被其他程序占用
  - 点"重新连接"重试

【设备列表为空】
  - 检查 USB 连接是否稳定
  - 在 iMouse 控制台确认设备"在线"
  - 重新插拔 USB 或点"刷新设备"

【发布失败：图标未匹配】
  - TikTok 更新后图标可能变化
  - 检查 icon/ 目录下对应的 .bmp/.png 是否过时
  - 工具会自动 fallback 到固定坐标

【视频上传卡住或连接中断】
  - 检查视频格式（必须 MP4/MOV，不支持 AVI/MKV/WMV）
  - 确认文件没被其他程序占用
  - 单文件超时 10 分钟，大视频请先压缩
  - 检查路径有没有包含特殊字符

【养号不工作 / 点错位置】
  - TikTok 界面更新后固定坐标可能失效
  - 检查 icon/heart.png 和 icon/home.png 是否过时
  - 横屏检测点的是 (71, 387)，如果 TikTok 改版这个坐标也要更新

【切号时"Switch account"点不到】
  - 确认已登录 2 个以上 TikTok 账号
  - 查看日志里的 OCR 识别结果
  - 横屏时会自动返回，但如果卡住请手动调整


═══════════════════════════════════════════════════════════════
  九、日志与调试
═══════════════════════════════════════════════════════════════

  - 底部黑色日志区显示所有操作历史
  - 颜色含义：
    绿色(ok)   — 成功
    红色(err)  — 错误
    黄色(warn) — 警告

  - 刷新设备后会打印第一台设备的原始字段用于调试
  - 发布/切号/养号每一步都会打印日志
  - 程序退出后日志不保存，有问题请先截图
